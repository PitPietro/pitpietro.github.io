Programming

Computer programming is the process of designing and building a program for performing a specific computing task. The purpose of programming is to find a sequence of instructions that will automate the performance of a task on a computer, usually for solving a given problem. Software engineering combines engineering techniques with software development practices. Reverse engineering is the opposite process.

There are different languages you can start whit, but the most important thing to keep in mind is the mental structure useful for facing and solving problems through programming. It is not important the language you talk, but the correctness with which you speak it.

A strong strategy is essential to avoid the so called spaghetti code and ensure an easy code maintenance. Otherwise, you’ll run into the following problems:

  1. Rigidity
    The difficulty in modifying the code and unknown development times.
  2. Fragility
    The tendency to malfunction.
  3. Immobility
    When parts of non-reusable code lead to duplication.
  4. Viscosity
    When a change leads to several changes.
To avoid those promlems, it's foundamental to implement the following paradigms and advice.

The 6 pillars

Object Orientated Programming

OOP stands for Object Orientated Programming. The Object Orientated paradigm is widely diffuse in all the most modern programming languages. C or Pascal doesn’t implement it, while Java, C++, C#, Object Pascal and even PHP do. This paradigm implies:

  1. Encapsulation
    It hides how a class does its business, while allows other classes to make requests to it.
  2. Abstraction & Generalizations
    Abstraction places the emphasis on what an object is (or does) rather then how it’s represent or how it works. Generalizations reduces complexity by replacing multiple entities which perform similar functions with a single construct.
  3. Inheritance
    It’s the ability of a class to extend another existing class (the so called super-class). A class can extend the functionality of it’s super-class by adding new types and method and by overriding existing ones.
  4. Polymorphism
    It’s the capability to perform the same operations by a wide range of different types of things. In OOP it’s achieved by:
    1. overloading
      The same method can be written multiple times whit different parameters in the same class.
    2. overriding
      A method inherited by a super class is modified by the extending class.

Source Control

Source control (or version control) is the practice of tracking and managing changes to code. The SCM (Source Control Management) provides a history of code and resolves conflicts during the merge of code from multiple contributors.

Git

Git is an open-source distributed source code management system. Git allows you to create a copy of your repository (branch). Using this branch, you can then work on your code independently from the stable version of your codebase. Once you are ready with your changes, you can store them as a set of differences (commit). You can pull in commits from other contributors to your repository, push your commits to others, and merge your commits back into the main version of the repository.

Design Patterns

Design Patterns are reusable solutions to commonly occuring problems. They are half cooked, which means that they aren’t really a code solution, but you will achieve it by using the logic behind the design pattern. The 23 main patterns are subdivided in three categories: Creatonal, Structural and Behavioral. I dedicated to them an entire page of my blog to avoid make this page too much confused.

Unit test

It’s a level of software testing where individual units (components) of a software are tested. The purpose is to validate that each unit of the software performs as designed. A unit is the smallest testable part of any software. It usually has one (or a few inputs) and usually a single output. In OOP, the smallest unit is a method. Unit testing frameworks, drivers, stubs, and mock/ fake objects are used to assist in unit testing.
